关于Coding与Git配合在Studio的一些使用

发表于 2016-11-15

1.Coding与Git的关联

  • 1:在coding.net上创建项目，用于放置你的项目的文件

    创建项目的基本选项 ：readme ignore

  • 2.确保studio中git 已经配置好

    打开idea，主菜单file-setting-version control-git 配置好git

  • 3.将项目加入git版本库

    在要提交的project项目文件夹初始化(不是 module)

  • 4.将本地库与远程库关联

    git remote add name url

  • 5.使用git之前都需要先更新过之后才能提交

    打开 pull 界面

    看到没有我们刚刚添加的remote url！点击右边的刷新按钮，然后选中webapp，这样就可以正常拉取代码了

  • 6. Push代码

    可以直接push 或者先 add 看一下都要push 那些代码。

    2.忽略文件

    Android Studio默认给出了三种忽略方式，供开发者进行选择（如上图红框）：

  • 1、忽略指定的文件
  • 2、忽略文件夹下所有文件
  • 3、忽略符合匹配规则的文件

    点击右侧的“+”进行添加，“-”删除。Androis Studio一般忽略以下不需要增加到版本库的文件：

  • 1、.idea 文件夹，此文件夹是用来保存开发工具的设置信息。
  • 2、 .gradle 文件夹，此文件夹是用来保存gradle的依赖信息。
  • 3、 所有的 build 文件夹，build文件夹是用来保存编译后的文件目录。
  • 4、 所有的 .iml 文件，是用来保存开发工具信息。
  • 5、local.properties 文件，是用来保存项目依赖信息。
